REACT 超入門 #04

入力フォーム
— 打った文字を State で覚える

前回の State の応用。今回は、あなたの予約アプリの「団体名」入力を例にやさしく。

前回(#03)の復習： State＝部品が持つ「変化する値」。 setCount で変えると画面が描き直される、でした。 今回はその「変わる値」が入力した文字になります。

1まず触ってみる

下の入力欄に文字を打つと、下にリアルタイムで反映されます。

入力中の文字：（まだ何も入力していません）
0 / 20 文字

この「今、入力欄に入っている文字」が State です。1文字打つたびに State が変わり、画面（反映表示・文字数・ボタンの押せる/押せない）も変わっています。

2React の考え方：入力欄の値＝State

「入力欄が文字を覚える」のではなく
React の State が文字を覚える。
入力欄はその State を映しているだけ。

ここが少し不思議なところ。ふつうは「入力欄に文字が溜まっていく」と考えますよね。React では逆で、本当の文字は State に入っていて、入力欄はそれを表示する鏡だと考えます。この作り方を 「制御コンポーネント」 と呼びます（名前は覚えなくてOK）。

32つの部品でできている

入力欄には、いつも次の2つをセットで書きます。

① value = 今の State を表示 ＋ ② onChange = 打つたび State を更新

この2つで「打つ → State更新 → 画面に反映」のループが回ります。#03 のカウンター（押す→State更新→再描画）と全く同じ仕組みです。

キーを打つ→ onChange が動く→ State が新しい文字に→ value に反映（再描画）

4コードで見る

やさしい形にするとこれだけです。

function Form() {
  const [name, setName] = useState("");   // 最初は空っぽの文字

  return (
    <input
      value={name}                              // ① 今の State を表示
      onChange={(e) => setName(e.target.value)}  // ② 打つたび更新
    />
  );
}
  • useState("") …… 最初は空文字の State を用意
  • value={name} …… 入力欄には「今の State」を映す
  • e.target.value …… 「今、入力欄に打たれた文字」のこと
  • setName(...) …… その文字で State を更新（→ 再描画）

5あなたのアプリでは（実物）

予約画面の「団体名」入力は、まさにこの形で書かれています👇

src/app/reserve/[date]/page.tsx
const [groupName, setGroupName] = useState("");
...
<input
  type="text"
  value={groupName}
  onChange={(e) => setGroupName(e.target.value)}
/>

さらに、この groupName を使って「団体名が空ならボタンを押せなくする」こともできます（上のデモのボタンと同じ挙動）。

// 団体名が1文字以上あるときだけ「次へ」を押せる
const canProceed = groupName.trim().length > 0;
...
<button disabled={!canProceed}>次へ進む</button>
つながった！
入力（State）→ その値でボタンの状態を決める。State を中心に画面ぜんぶが決まる、という React の考え方が、あなたのアプリで実際に動いています。

!よくあるつまずき

value だけ書いて onChange を忘れる
<input value={name} /> だけだと、文字を打っても何も入力できません（State が変わらないから、鏡がずっと同じものを映し続ける）。value と onChange は必ずセット。

★用語ミニ辞典

制御コンポーネント
入力欄の値をState で管理する作り方。React の基本スタイル。
value
入力欄に「今表示する文字」。State を指定する。
onChange
入力欄の文字が変わった瞬間に動く合図。ここで State を更新する。
e.target.value
「今、入力欄に入っている文字」を取り出す書き方。

✓今日のまとめ

  • 入力フォームは「入力欄の値＝State」で作る（制御コンポーネント）。
  • value（今の State を表示）と onChange（打つたび更新）を必ずセットで書く。
  • 仕組みは #03 のカウンターと同じ：打つ→State更新→再描画。
  • State の値で「ボタンを押せる/押せない」なども決められる（disabled）。
  • あなたの予約アプリの団体名入力が、まさにこの形。
