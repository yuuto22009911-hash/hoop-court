REACT 超入門 #08 — 総まとめ

全部つなげて、1画面を読む

#00〜#07 で学んだ道具が、1つの画面でどう組み合わさるか。バラバラの知識が「腑に落ちる」回です。

今回のゴール： 新しいことは学びません。これまでの7つの道具が、簡単な予約画面の中で どう協力しているかを色分けして読むだけ。読めたら、React の基礎は卒業です。

17つの道具・早見表

#道具ひとことで
01コンポーネント画面を返す関数（部品）
02Props親→子へ渡すデータ（読むだけ）
03State変化する値。変えると再描画
04入力value＋onChangeで State に結ぶ
05リスト/map配列→部品の並び（key必須）
06条件表示? : / && / 早期return
07useEffect画面表示後に自動実行（データ取得）

2これを読み解く（色が道具の対応）

簡単にした予約画面です。右の色タグが「どの道具か」を示します。

簡略版：予約画面 ReservePage.jsx
function ReservePage() {                            01 部品

  const [slots, setSlots] = useState([]);          03 State
  const [groupName, setGroupName] = useState("");   03/04

  useEffect(() => {                                07 useEffect
    fetchSlots().then((data) => setSlots(data));   // 開いたら取得
  }, []);                                          // 最初の1回だけ

  const canProceed = groupName.trim().length > 0;  // 派生値

  return (
    <div>
      {slots.map((s) => (                          05 リスト
        <SlotButton key={s.id} time={s.time} ok={s.ok} />  02 Props
      ))}

      <input                                      04 入力
        value={groupName}
        onChange={(e) => setGroupName(e.target.value)}
      />

      {canProceed                                  06 条件
        ? <button>次へ進む</button>
        : <p>団体名を入力してください</p>}
    </div>
  );
}

function SlotButton(props) {                        01 部品
  return (
    <button>
      {props.time} — {props.ok ? "◎" : "×"}      06 条件 02 Props
    </button>
  );
}

31行ずつ、日本語で読む

  • 01 ReservePage と SlotButton は部品（画面を返す関数）。大きい部品の中に小さい部品を置く。
  • 03 slots（空き枠）と groupName（入力文字）はState。変わると画面が描き直される。
  • 07 useEffect が「開いた瞬間」に fetchSlots() でデータ取得 → setSlots で State に入れる。
  • 05 slots.map(...) で、空き枠の数だけ SlotButton を並べる（key付き）。
  • 02 各 SlotButton に time と ok をPropsで渡す。子はそれを表示するだけ。
  • 04 input は value＋onChange で groupName に結ばれた入力欄。
  • 06 canProceed ? ... : ... と props.ok ? "◎" : "×" が条件表示。
気づけましたか？
7回に分けて学んだ道具が、たった30行の中に全部同居しています。実際のアプリはこれが大きくなっただけ。読める道具が増えれば、大きなコードも「知ってる部品の集まり」に見えてきます。

4ユーザー操作から画面更新までの一巡

「団体名を入力する」を例に、道具がどうバトンを渡すか。

1ページを開く → useEffect(07) が空き枠を取得 → State(03) 更新
2map(05) が枠の数だけ SlotButton(01) を並べ、Props(02) で時刻と◎/×を渡す
3ユーザーが団体名を入力 → onChange(04) が State(03) groupName を更新
4State が変わったので React が再描画 → canProceed を計算し直す
5条件表示(06) が「次へ進むボタン」か「入力を促す文」かを出し分ける

この「操作 → State更新 → 再描画 → 表示が変わる」のループこそ、#00 で言った 「UI ＝ f(state)」 の正体です。全部つながりました。

✓基礎編・卒業チェック

下の文がスッと読めたら、React の基礎はOKです👏

  • 画面は部品(コンポーネント)の組み合わせで作る。
  • 部品にはPropsで外からデータを渡す（子は読むだけ）。
  • 変化する値はState。setXxxで変えると再描画される。
  • 入力欄は value＋onChange で State に結ぶ。
  • 繰り返しは map＋key、出し分けは ? :／&&／早期return。
  • 外とのやりとりは useEffect（開いたら取得など）。
