REACT 超入門 #01

コンポーネントって何？
— 画面を「部品」で作る

前回の続き。今回も専門用語はやさしく、たとえ話と図でいきます。

前回(#00)の復習： React は「データを変えたら、画面を自動で書き換えてくれる道具」でした。 今回はもう一つの大事な役目 ——「画面を部品で作る」の話です。

1コンポーネント＝画面の「部品」

まず結論。

React では、画面を1枚の大きな絵で描くのではなく
小さな「部品」を組み合わせて作る。
この部品を コンポーネント と呼ぶ。

ふだん見ているアプリの画面も、よく見ると「部品の集まり」です。下の図のように分けられます👇

見出しバー・カード・ボタン・下のタブ —— この一つひとつがコンポーネント（部品）です。

小さなブロックを作っておけば、同じブロックを何度でも使い回せるのがミソ。 「ボタン」を1回作れば、あちこちで <Button /> と置くだけで同じボタンが何個でも出せます。

3実際のコンポーネントを見てみる

身構えなくて大丈夫。「画面を返す関数」というだけです。

いちばん簡単なコンポーネントはこんな形。「部品の名前」と「表示する中身」を書いた関数です。

function Button() {          // ← Button という名前の部品
  return <button>予約する</button>;   // ← 画面に出す中身
}

あとは、使いたい場所にタグのように置くだけ👇

<Button />   // ここに「予約する」ボタンが出る
<Button />   // もう1個置けば、同じボタンがもう1個出る（使い回し！）
小さな決まりごと
部品の名前は大文字はじまり（Button）。小文字だと、React が「普通の HTML タグ」だと勘違いします。今は「へぇ」で OK。

4部品を組み合わせて画面を作る

部品の中に、別の部品を置けます。これで画面が完成します。

function App() {         // ← 画面ぜんぶ、を表す部品
  return (
    <div>
      <Header />    // 見出しバー
      <DayCard />   // カード
      <Button />    // ボタン
      <TabBar />    // 下のタブ
    </div>
  );
}

5部品にすると何がうれしい？

① 使い回せる
同じ部品を何度も書かなくていい。1回作れば置くだけ。
② 分けて考えられる
「今日はボタンだけ直す」と、部品ごとに集中できる。大きな画面を一気に相手にしなくていい。

★用語ミニ辞典

コンポーネント
画面の「部品」。中身は「画面を返す関数」というだけ。例：ボタン、カード、見出しバー。
関数
「何かを受け取って、何かを返す機械」。コンポーネントは「呼ばれたら"画面"を返す関数」。
タグ
<Button /> のような書き方。「ここにこの部品を置く」という意味。

✓今日のまとめ

  • React は画面を「部品（コンポーネント）」の組み合わせで作る。
  • コンポーネントの正体は「画面を返す関数」。難しく考えない。
  • 部品は使い回せる（1回作って何度も置ける）。
  • 部品ごとに分けて考えられるので、複雑な画面もラクになる。
