REACT 超入門 #02

部品にデータを渡す（Props）
— 同じ部品で、違う中身を出す

前回の「同じ部品を使い回せる」の続き。今回も、たとえ話と図でやさしくいきます。

前回(#01)の復習： コンポーネント＝画面の「部品」。正体は「画面を返す関数」で、 <Button /> と置けば何個でも使い回せる、でした。

1こんな疑問、ありませんか？

前回、同じボタンを何個でも置ける、と言いました。でも実際のアプリでは…

予約する キャンセル

のように文字が違うボタンが要りますよね。「予約する」用と「キャンセル」用で、別々の部品を作らないといけないの？

いいえ。同じ部品に 「文字だけ」外から渡せば OK。
これが Props（プロップス）。

2Props＝部品に渡す「具」

たとえ話：たい焼きの型 🐟

🫘

予約するボタン
  • 部品（型）はいつも同じ形 …… Button
  • Props（具）を変えると中身が変わる …… "予約する" / "キャンセル"

型は1つでいい。入れる具（Props）を変えるだけで、違うボタンになります。

3コードで見る

「受け取る側」と「渡す側」の2つがあります。

① 受け取る側（部品の定義） — props という箱で具を受け取る：

function Button(props) {
  return <button>{props.label}</button>;
  //               ↑ 受け取った文字をここに表示
}

② 渡す側（使うところ） — label= で具を渡す：

<Button label="予約する" />     // →「予約する」ボタン
<Button label="キャンセル" />   // →「キャンセル」ボタン
読み方
label は「名前（好きに付けてよい）」。{props.label} は 「受け取った label をここに出す」という意味。{ } は「ここに JavaScript の値を入れる窓」でしたね（#00の応用）。

4データは「親 → 子」へ流れる

Props は、外側の部品（親）から中の部品（子）へ一方通行で渡されます。

親：<Button label="予約する" /> ──渡す──▶ 子：Button の中で props.label を表示
大事なルール
渡された Props は、子の側で勝手に書き換えてはいけません（読むだけ）。 「もらった具はそのまま使う。作り変えない」と覚えておけば十分です。

5文字以外も渡せる

Props で渡せるのは文字だけではありません。色・数字・オン/オフなど何でも渡せます。

<Button label="予約する" color="blue" />
<DayCard date="7/3" available={true} />   // 空きあり
<DayCard date="7/5" available={false} />  // 満員

同じ DayCard でも、date（日付）と available（空きの有無）を変えるだけで、カレンダーの各日を表示できます。あなたの体育館予約アプリのカレンダーも、この発想で作られています。

★用語ミニ辞典

Props（プロップス）
部品に外から渡すデータ（設定・具）。properties（性質）の略。
親・子
部品を「置く側」が親、「置かれる側」が子。Props は親→子へ流れる。
読み取り専用
子は Props を見る（使う）だけ。書き換えてはいけない、という約束。

✓今日のまとめ

  • Props＝部品に外から渡すデータ。たい焼きの「具」。
  • 同じ部品でも、Props を変えれば違う中身を表示できる。
  • 渡し方は <Button label="予約する" />、受け取りは props.label。
  • Props は親→子の一方通行。子は書き換えず、読むだけ。
