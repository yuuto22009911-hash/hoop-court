REACT 超入門 #00

そもそも React って何？
— ゼロから、やさしく。

専門用語はできるだけ使わず、使うときは必ず説明します。まずは「React とは何者か」だけをしっかり理解しましょう。

1一番大事な結論

最初にこれだけ覚えてください。

React は「言語」ではありません。
JavaScript という言語を、もっと楽に使うための
「道具セット（ライブラリ）」です。
たとえるなら
「英語」が言語、「電子辞書」が道具。
React を使うときも、書いているのは JavaScript という言語そのものです。

2Web ページは「3つの部品」でできている

ここを押さえると、React の居場所がハッキリします。

ホームページ（Web ページ）は、役割の違う3つの部品でできています。

部品役割家にたとえると
HTML文字・ボタン・画像を置く（骨組み）柱・壁
CSS色・大きさ・配置など見た目壁紙・ペンキ
JavaScriptクリックで動く・数字が変わるなど動き電気・水道の仕組み
ポイント
この3つのうち JavaScript が「言語」。ボタンを押したら表示が変わる、といった「動き」を担当します。
そして React は、その JavaScript を助ける道具です。

3React は「動き」を楽にする道具

React が生まれた理由＝解決したい面倒。

JavaScript だけで「動き」を作ると、こんな面倒があります。

例：数字が増えるカウンターを作る
「数字が変わった → 画面のどの文字を、どうやって書き換えるか」を、 全部自分の手で指示しないといけない。ページが複雑になると、この指示が爆発的に増えてぐちゃぐちゃに…。

React はここを自動化します。

データを更新（数字は 5 です） → 画面のどこを書き換えるかは React が自動でやる → 画面が勝手に追従
あなたは「データを変えるだけ」。
画面の書き換えは React にお任せ。

4たとえ話：料理

JavaScript だけ
材料を切って、火加減を見て、盛り付けまで全部自分で手作業。
React を使う
「今日はカレー」と決めるだけで、手順を段取りしてくれる調理アシスタントがいる。

あなたは「何を作るか（＝どんなデータか）」を決めるだけ。細かい手順は React が引き受けてくれる、というイメージです。

5なぜみんな React を使うの？

  • 楽になる：画面の書き換えを手作業でやらなくていい。
  • 部品にできる：ボタンやカードを「部品」として作り、あちこちで使い回せる（次回以降のテーマ）。
  • 仲間が多い：世界中で使われていて、困ったとき情報がすぐ見つかる。あなたの hoop-court アプリも React 製です。

★用語ミニ辞典

言語（プログラミング言語）
コンピュータへの命令を書くための言葉。ここでは JavaScript がそれ。
ライブラリ
誰かが作ってくれた「便利な道具の詰め合わせ」。React はライブラリ。自分で一から作らず、これを借りて楽をする。
JavaScript（JS）
Web ページに「動き」をつける言語。React はこの JavaScript の上で動く。
HTML / CSS
HTML＝骨組み（何を置くか）、CSS＝見た目（色や大きさ）。

✓今日のまとめ（これだけ持ち帰る）

  • React は言語ではなく、JavaScript を楽にする道具（ライブラリ）。
  • Web ページは HTML（骨組み）／ CSS（見た目）／ JavaScript（動き） の3部品。
  • React の役目は「データを変えたら、画面を自動で書き換える」こと。
  • あなたはデータを決めるだけ、細かい画面更新は React に任せる。
