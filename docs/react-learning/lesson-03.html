REACT 超入門 #03

画面が「変化」する仕組み（State）
— ボタンを押すと数字が増える

いよいよ React の心臓部。ここが分かると「React 便利!」が実感できます。今回もやさしく。

これまで： #00 React は道具 / #01 画面は部品（コンポーネント）/ #02 部品にデータを渡す（Props）。 今回は「画面が動いて変わる」仕組みです。

1まず、実際に触ってみる

下のボタンを押してみてください（このページ内で本物が動きます）。

0今の数字
押すたびに数字が増えます。この「変わる数字」が State です。

「押す → 数字が増える → 画面の表示も変わる」。この変化する値のことを、React では State（ステート＝状態）と呼びます。

2State＝部品が"覚えておく"変わる値

State ＝ 部品の中で変化して覚えておくデータ。
State が変わると、React が画面を自動で描き直す。

#00 で言った「データを変えたら画面が自動で書き換わる」の「データ」がまさにこの State です。ここで全部つながります。

ボタンを押す → State（数字）が 0 → 1 に変わる → React が画面を描き直す → 表示が「1」になる

3Props と何が違う？

Props（前回 #02）
外からもらう。
子は変えられない（読むだけ）。
例：ボタンの文字。
State（今回 #03）
部品が自分で持つ。
変えられる（変えると再描画）。
例：押した回数。

ざっくり：Props＝外からもらう固定の設定、State＝自分で変えられる中身。

4コードで見る（useState）

State は useState という道具で作ります。

function Counter() {
  const [count, setCount] = useState(0);
  //     ↑今の値   ↑変える関数        ↑最初は 0

  return (
    <button onClick={() => setCount(count + 1)}>
      {count} 回
    </button>
  );
}

読み方：

  • useState(0) …… 「最初は 0 の State を作って」
  • count …… 今の値（表示に使う）
  • setCount …… 値を変える専用ボタン（関数）
  • onClick={() => setCount(count + 1)} …… クリックで「今の値＋1」に更新
セット（2つ組）で受け取る
const [count, setCount] = useState(0) は「今の値」と「変える関数」を1行で受け取る決まり文句。名前は自由ですが xxx / setXxx にするのが慣習です。

5いちばん大事な約束

やってはいけない
count = count + 1 と直接書き換えても画面は変わりません。 React が「変わった」と気づけないからです。
正しいやり方
必ず setCount(...)（変える関数）を通す。これを呼ぶと React が 「State が変わった → 描き直そう」と動いてくれます。
State を変えるときは
必ず「set○○」の関数を使う。

★用語ミニ辞典

State（ステート／状態）
部品が持っていて、変化する値。変えると画面が描き直される。例：押した回数、入力中の文字、開いてる/閉じてる。
useState
State を作る道具。[今の値, 変える関数] = useState(最初の値) の形で使う。
再描画（再レンダリング）
State が変わったとき、React が画面を自動で書き直すこと。

✓今日のまとめ

  • State＝部品が持つ「変化する値」（押した回数など）。
  • State が変わると React が画面を自動で描き直す。#00 の「データを変えたら画面が変わる」の正体。
  • 作る道具は useState。[count, setCount] = useState(0)。
  • 変えるときは必ず set○○ 関数を使う（直接書き換えない）。
  • Props＝外からもらう固定の設定 / State＝自分で変えられる中身。
