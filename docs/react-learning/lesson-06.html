REACT 超入門 #06

状況で表示を出し分ける
— 条件付き表示

「空きなら◎・満員なら×」「ログイン中は名前・未ログインはボタン」——こういう出し分けの書き方です。

これまで： #01 部品 / #02 Props / #03 State / #04 入力 / #05 リスト。 今回は「状況に応じて見せるものを変える」テクニックです。

1まず触ってみる

下のボタンで状態を切り替えると、画面（下の枠）の表示が変わります。

同じ画面でも、状態（State）によって中身が丸ごと変わりました。これが条件付き表示です。#03 の State と組み合わせて使います。

2考え方：if で見せ方を変えるだけ

「もし◯◯なら A を見せる、
そうでなければ B を見せる」——
ふだんの if の考え方を、そのまま画面に使う。

難しい新概念はありません。状況（State や Props）を見て、返す JSX を変えるだけ。書き方（イディオム）が3つあるので順に紹介します。

3書き方①：三項演算子（A か B か）

「2択」のときに使う定番。条件 ? A : B の形です。

{isAvailable ? "◎ 空き" : "× 満"}

// 部品まるごとの2択もできる
{isLoggedIn ? <p>ようこそ{name}さん</p> : <button>ログイン</button>}

読み方：「isAvailable が true なら ◎ 空き、false なら × 満」。上のデモの◎/×もこれで作れます。

4書き方②：&&（あるときだけ見せる）

「条件を満たすときだけ表示、満たさなければ何も出さない」ときは 条件 && A。

// 選択済みのときだけ「選択中」バーを出す
{hasSelection && <p>選択中: 9:00〜10:00</p>}

// 金額が決まったときだけ表示
{amount > 0 && <p>金額: {amount}円</p>}

「else が要らない」ときはこちらがスッキリします。あなたの予約画面でも、選択中バーや金額表示にこの形が使われています。

つまずきポイント
{count && <p>...</p>} のように数字を左に置くと、count が 0 のとき画面に「0」がそのまま出てしまいます。 amount > 0 && のように必ず true/false になる形にしましょう。

5書き方③：早期 return（画面まるごと切替）

状態ごとに画面が大きく違うときは、先に return して抜けるのが読みやすいです。

src/components/LiffGate.tsx（考え方は同じ・簡略化）
if (state === "loading")      return <p>読み込み中...</p>;
if (state === "unregistered") return <p>登録画面に移動...</p>;
return <>{children}</>;   // どれにも当てはまらなければ本編を表示

上から順に「当てはまったらそれを返して終わり」。あなたのアプリの LiffGate は、まさにこの形で「読み込み中 → 未登録なら登録画面 → 登録済みなら本編」を出し分けています。

63つの使い分け

A ? B : C
2択。どちらかを必ず見せる。◎/× など。
条件 && A
あるときだけ見せる。else 不要。
早期 return
画面が丸ごと違うとき。状態が3つ以上でも読みやすい。

迷ったら：2択なら ?: 、出す/出さないなら && 、大きく違うなら早期 return。

★用語ミニ辞典

条件付きレンダリング
状況（State/Props）に応じて表示する中身を変えること。「レンダリング」＝画面に描くこと。
三項演算子 ? :
条件 ? A : B。条件が true なら A、false なら B。2択の省略 if。
&&
条件 && A。条件が true のときだけ A を表示。false なら何も出さない。
早期 return
条件に合ったら、その場で JSX を return して関数を抜ける書き方。

✓今日のまとめ

  • 条件付き表示＝状況を見て、返す JSX を変えるだけ。新概念ではなく「if の応用」。
  • 2択は 条件 ? A : B（例：◎/×）。
  • 出す/出さないは 条件 && A（数字を左に置かない！）。
  • 画面まるごと違うときは早期 return（あなたの LiffGate）。
  • State（#03）と組み合わせると「押す→状態が変わる→表示が切り替わる」が作れる。
