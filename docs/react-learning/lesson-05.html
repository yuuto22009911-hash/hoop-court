REACT 超入門 #05

たくさん並べて表示する
— リストと map

カレンダーの日付や時刻表のような「同じ形の繰り返し」を、コピペせずに作る方法です。

これまで： #01 部品 / #02 Props / #03 State / #04 入力フォーム。 今回は「同じ部品をデータの数だけ並べる」話です。

1こんなの、手で書きたくないですよね

予約カレンダーには日付がたくさん並びます。これを全部手書きすると…

<DayCard date="7/3" />
<DayCard date="7/4" />
<DayCard date="7/5" />
... 30個 …もう嫌になる 😵

データが増えるたびに書き足すのは大変だし、間違いのもと。データの一覧から自動で並べたいですよね。

2map ＝「一覧」を「部品の一覧」に変える

データの配列を .map() に通すと
部品（JSX）の並びに一気に変換できる。

.map()（マップ）は「配列の1個1個を、別のものに作り変える」JavaScript の道具。React では「データ1個 → 部品1個」に変換するのに使います。

データの配列
"7/3" "7/4" "7/5"
.map() ▶
部品の並び
<DayCard 7/3> <DayCard 7/4> <DayCard 7/5>

3コードで見る

配列を用意して、.map() で部品に変換するだけ。

const dates = ["7/3", "7/4", "7/5"];   // データの一覧

return (
  <div>
    {dates.map((date) => (
      <DayCard key={date} date={date} />
    ))}
  </div>
);
  • dates.map((date) => ...) …… 配列の1個ずつ（date）を受け取り、<DayCard> に変換
  • 結果は <DayCard> が3個並んだもの
  • データを増やせば、書き足さなくても勝手に増える👍

4key ＝ 一個ずつに付ける「名札」

気づきましたか？ さっきのコードに key={date} がありました。並べるときは、各部品に key（名札）を必ず付けます。

key ＝ 一個ずつを見分けるための名札。
同じものが重複しない値を付ける。

なぜ必要？ — React が「どれがどれ」を見分けるため。

たとえ話
クラスの名簿で全員「名無し」だと、1人転校しても誰が抜けたか分からない。 名札（key）を付けておけば、React は「7/4 のカードはそのまま、7/5 だけ満員に変わった」と正確に・効率よく画面を更新できます。
key に選ぶもの
日付やID など「重複しない値」を使う。上の例では日付そのもの（key={date}）が名札になっています。

5あなたのアプリでは（実物）

予約画面の「時刻表」は、まさに map で作られています👇（少し形は違いますが考え方は同じ）

src/app/reserve/[date]/page.tsx
{Array.from({ length: SLOTS_PER_DAY }, (_, i) => i).map((i) => (
  <button key={i} onClick={() => onTapSlot(i)}>
    {minToHHMM(...)} 〜 {minToHHMM(...)}
  </button>
))}
  • 「30分スロットの番号（0,1,2,…）」という配列を作り、.map() でボタンに変換
  • key={i} …… スロット番号を名札に（重複しないのでOK）
  • これで「9:00〜9:30」「9:30〜10:00」…と時刻ボタンが自動でずらっと並ぶ

!よくあるつまずき

key を付け忘れる
{list.map(x => <li>{x}</li>)} のように key なしで並べると、 「key が無いよ」と警告が出ます。表示はされますが、後でリストが変わったとき表示が崩れるバグの原因に。必ず付けましょう。
JSX の中では for 文が書けない
「繰り返し」は for ではなく .map() で書く、が React の作法です。

★用語ミニ辞典

配列
データを順番に並べて入れる箱。["7/3", "7/4", "7/5"] のような形。
.map()（マップ）
配列の1個ずつを別のものに作り変えて、新しい配列を返す道具。React では「データ→部品」に使う。
key（キー）
並べた部品1個ずつに付ける名札。React が要素を見分けるために必要。重複しない値にする。

✓今日のまとめ

  • 同じ形の繰り返しは、手書きせず 配列を .map() で部品に変換して並べる。
  • {dates.map((date) => <DayCard key={date} date={date} />)} が基本形。
  • 並べた部品には key（名札）を必ず付ける。重複しない値で。
  • データが増えれば表示も自動で増える。あなたのアプリの時刻表もこれ。
