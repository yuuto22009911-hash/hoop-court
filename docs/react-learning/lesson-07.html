REACT 超入門 #07

外の世界とやりとりする
— useEffect

「ページを開いたら、サーバーへデータを取りに行く」——こういう処理の置き場所です。今回もやさしく。

これまで： 画面が動くきっかけは「ユーザーの操作」（クリック・入力）でした。 でも今回は「操作していないのに、画面が出た瞬間にやりたいこと」の話です。

1こんなこと、したくないですか？

  • 予約ページを開いたら、その日の空き状況をサーバーから取ってくる
  • アプリを開いたら、LINE ログインを初期化する
  • マイページを開いたら、自分の予約一覧を読み込む

どれも「ボタンを押したら」ではなく、「画面が表示されたら自動で」やりたい処理です。これを書く場所が useEffect（ユーズ・エフェクト）です。

2useEffect＝「画面を出したあとに、これやっといて」

useEffect ＝ 画面を描き終わったあとに
自動で実行してほしい処理を書く場所。

React は「まず画面を出す」→「そのあと useEffect の中身を実行」という順で動きます。だから「開いた瞬間にデータ取得」がここに書けるのです。

1ページを開く → React が画面を描く（まず「読み込み中...」を表示）
2useEffect が動く → サーバーへデータを取りに行く
3データが返ってきたら State を更新（#03）→ 画面が描き直される
4「読み込み中...」が、取れたデータの表示に変わる
たとえ話
お店を開けたら（画面表示）、まず倉庫に在庫を確認しに行く（useEffect でデータ取得）。 在庫が分かったら棚に並べ直す（State更新→再描画）。

3触ってみる

ボタンを押すと「ページを開いた」ことにして、データ取得（1.2秒の待ち）を再現します。

まだ開いていません

「開く → まず読み込み中 → 少し待つ → データ表示」。この流れが useEffect の典型です。

4コードで見る

形はいつも useEffect(() => { やりたいこと }, [いつ動かすか])。

const [slots, setSlots] = useState([]);   // 空き状況を入れる箱

useEffect(() => {
  // 画面が出たあとに自動で動く
  availabilityRange(...).then((r) => {
    setSlots(r.slots);   // 取れたデータで State を更新 → 再描画
  });
}, []);                    // ← [] = 最初の1回だけ動かす
  • 1つめの () => { ... } …… やりたい処理（データ取得など）
  • 2つめの [] …… いつ動かすかを指定する「依存配列」

5いちばん大事：2つめの [ ]（依存配列）

「いつ動かすか」を、この配列で決めます。ここが useEffect の肝です。

[]（空）
最初の1回だけ動く。
例：開いたときにログイン初期化。
[date]
date が変わるたびに動く。
例：見ている日付が変わったら空き状況を取り直す。
イメージ
「ここに入れた値が変わったら、もう一度やってね」というお願いリスト。 空っぽ [] なら「変わるものが無い＝最初の1回だけ」。

6あなたのアプリでは（実物）

予約画面では、日付が変わるたびに空き状況を取り直しています👇

src/app/reserve/[date]/page.tsx（簡略化）
useEffect(() => {
  availabilityRange(courtId, from, to)
    .then((r) => setSlots(r.slots));   // 取得→State更新
}, [courtId, date]);   // コートか日付が変わったら取り直す

LiffGate でも、開いたときに initLiff()（LINEログイン初期化）→ authMe()（本人確認）を useEffect でやっています。「開いた瞬間に外とやりとり」の典型例です。

!軽く知っておく：後片付け（クリーンアップ）

useEffect は、return で「後片付けの処理」も書けます。画面を閉じるときに呼ばれます（タイマー停止など）。

useEffect(() => {
  const id = setInterval(...);      // タイマー開始
  return () => clearInterval(id);   // ← 閉じるとき止める（後片付け）
}, []);
今は「そういうのもある」でOK
「開いたとき何かを始めたら、閉じるとき止める」——それを書く場所が return 部分、とだけ覚えておけば十分です。あなたの LiffGate にもこの後片付けが入っています。

★用語ミニ辞典

useEffect
画面を描いたあとに自動で動かしたい処理を書く道具。データ取得・初期化など。
副作用（effect）
「画面を返す」以外の処理（通信・タイマーなど）のこと。effect の名前の由来。
依存配列 [ ]
useEffect を「いつ動かすか」を決める2つめの引数。[]＝最初の1回、[x]＝x が変わるたび。
クリーンアップ
useEffect の return に書く「後片付け」。画面を閉じるときに動く。

✓今日のまとめ

  • useEffect＝画面を描いたあとに自動で動かしたい処理の置き場所。
  • 典型は「開いたらデータを取りに行く」。取れたら State 更新（#03）で画面に反映。
  • 形は useEffect(() => { 処理 }, [いつ])。
  • 2つめの 依存配列で動くタイミングを決める：[]＝1回だけ / [x]＝x が変わるたび。
  • return に後片付け（クリーンアップ）を書ける。
  • あなたのアプリの空き状況取得・LINEログイン初期化がまさにこれ。
